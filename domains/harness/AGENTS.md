# harness

Назначение: Evidence ingestion: watchers, probes, runtime health, browser signals и repo facts.

Когда идти сюда:
- Когда система учится слушать проект и превращать его в говорящую среду.

Соседи: git, browser, artifact, verify

Карта:
```text
harness
├─ AGENTS.md
└─ docs/
   └─ context.mmd
```

Текущее состояние:
- Материализована карта домена.
- Код появится по owning slices, когда домен станет активным фронтом реализации.
