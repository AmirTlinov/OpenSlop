# PHILOSOPHY

## Зачем существует OpenSlop

OpenSlop нужен для одной вещи: дать человеку по-настоящему нативный, быстрый и честный macOS workbench, где можно использовать не одного вендора, а любых нужных агентных исполнителей под одним GUI.

Этот продукт рождается из раздражения двумя проблемами:
1. хорошие агентные движки живут в разрозненных интерфейсах;
2. даже красивые GUI часто превращают реальную разработку в тёмную комнату без проверяемых сигналов.

## Что мы строим

Мы строим local-first Agent Workbench:
- нативное macOS приложение;
- единый рабочий центр для Codex, Claude Code и следующих providers;
- platform-owned browser, harness, verify, review и artifact surfaces;
- fail-closed окружение, где проект говорит фактами, а не самоотчётами.

## Непереговорные принципы

### 1. Native first
Продукт должен ощущаться как настоящий Mac-продукт. Его форма рождается из macOS, а не из браузерной IDE в нативной рамке.

### 2. Provider plurality
Никакой vendor lock-in как центральная идея. Сильные стороны каждого provider сохраняются через capability model.

### 3. UI не источник истины
Окно можно закрыть. Runtime и сессии продолжают жить. Истина принадлежит core daemon и event spine.

### 4. Fail-closed by evidence
`UNKNOWN` не считается `PASS`. Если проект не доказал здоровье, интерфейс должен говорить это прямо.

### 5. Scale by architecture
Сотни активных и архивных сессий должны оставаться рабочими. Производительность закладывается в хранение, проекции, виртуализацию и шумоподавление.

### 6. Human legibility
Сильный продукт не прячет реальность за магией. Пользователь должен понимать, что происходит, почему это произошло и что доказано.

## Что мы сознательно не строим

- GUI-обёртку над терминалом;
- общий знаменатель, который делает всех агентов одинаково слабыми;
- chat-first продукт без review, verify и browser truth surfaces;
- монолитную архитектуру, где всё течёт через один god-object.

## Как выглядит успех

У пользователя открыт один workbench. В нём живут десятки и сотни сессий. Он может переключаться между проектами, запускать Codex и Claude рядом, видеть diff, preview, browser trace, verify-gates и approvals, и всё это не разваливается на слайдшоу.
